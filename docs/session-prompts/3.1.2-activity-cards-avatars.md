## 关键决策（先想清楚再写）

### 1. 数据接口怎么走

当前 `/api/get-activities` 不返回 RSVP。三个候选：
- (a) 扩展 `/api/get-activities` 加可选 `?include_rsvps=1` flag
- (b) 写新 `/api/activities-with-rsvps` 端点
- (c) 客户端先拉 `/api/get-activities` 再 fetch `/api/rsvps?activity=xxx`

**推荐 (a)** + 复用 `_rsvp.js` 的 `fetchAllRsvps`（KV 缓存了），server 端 join 一次，避免 N+1。

注意：移动端流量敏感，每张卡只回前 N 个头像（嘉宾 5 / 报名 8），但要带 total 计数显示 "+N"。

### 2. Modal 形态

按 [[homepage-design]] 第 5 节"开放问题"我们倾向 **C（modal popup）—— 最简单、不破坏布局**。

桌面：居中半屏 modal；移动：底部 sheet 滑入。

### 3. 头像图片 source

成员表已有 `avatar_token` 字段，前端拼 `/api/poster?token=xxx`。沿用 `api/members.js` 已建的模式。
