# cyc.center 竞品分析（Phase 2.2）

> 用 3 个 persona（玖玖 / Alex / Mia）作为评估维度。问每个竞品："对每个 persona 它做对了什么、做错了什么？cyc 该偷什么、避免什么？"
>
> **核心定位锚点**：[Selina × Notion] —— 实体社区 + 数字 OS，可复用到其他空间。

---

## A 组 — 实体 coliving / 社区空间（漏斗对标）

### A1. DNA 数字游民公社（安吉）🇨🇳

**一句话**：国内最知名的数字游民集中营，废弃竹木厂改造，2500 万投资。

| | 对玖玖 | 对 Alex | 对 Mia |
|---|---|---|---|
| 吸引点 | ❌ 不感兴趣（游民标签她抵触）| ✅ 强（"我同类在这里"）| ⚠️ 看一眼但不会去（远）|
| 转化路径 | — | A→B 完整，价格透明（300-2000/月）| — |
| 留存 | — | 中等，主要靠"我们 70 人"群体感 | — |

**做对了**：
- 价格梯度透明（双人 300+ / 单间 ≤2000）—— Path B 教科书
- "废弃竹木厂改造"故事好讲 —— Path A 有 origin story
- 联合发起人有大理脉络（阿德、许崧）—— 给 cyc.center 反向参考

**做错了**：
- 重押"游民"单一身份 —— 排他性强（玖玖型本地人不进来）
- 网站偏机构风，缺人格化
- 留存机制弱 —— 游民走了就走了，没有"远程关注"通道

**cyc 该偷**：🟢 价格透明梯度（Path B）· origin story 叙事
**cyc 该避免**：🔴 单一身份标签（cyc 应同时容纳 玖玖 + Alex + Mia）

---

### A2. Outsite 🌍

**一句话**：全球远程工作者高端 coliving 网络，会员制，月费 ~$150。

| | 对玖玖 | 对 Alex | 对 Mia |
|---|---|---|---|
| 吸引点 | ❌ 无（不在大理）| ✅ 高（专业 + 全球） | ❌ 太正式 |
| 转化路径 | — | A→B 流畅（locations / pricing 透明）| — |
| 留存 | — | 强（会员 perks + 全球 portability）| — |

**做对了**：
- 网站像 boutique hotel × LinkedIn —— 专业感强但不冷冰
- **Membership-first business model**：$150/月先付，住房再 40% off —— 分离了"成为成员"和"住"
- Outsite Curated（独立空间策展）—— 平台化思维
- App + 跨地点 portability —— 给 Alex 这种"试三个月看看"的人持续粘性

**做错了**：
- 太"产品化"，缺地方文化 —— 每个 Outsite 都长得差不多
- 价格门槛排除了非专业人士

**cyc 该偷**：🟢 **Membership ≠ 居住** 这个分离 —— 你可以是 cyc 成员但不住在 cyc · 房型/地点页面的清晰布局
**cyc 该避免**：🔴 千篇一律的视觉模板（你的"大理玻璃"刚好是反面）

---

### A3. Selina 🌎

**一句话**：南美起家的 hostel + coliving + coworking 三合一，明显 boho-luxe 嬉皮风。

| | 对玖玖 | 对 Alex | 对 Mia |
|---|---|---|---|
| 吸引点 | ❌ 调性不对（嬉皮 vs 创意人）| ⚠️ 中（视觉炫但有点 generic）| ✅ 强（拍照好看）|
| 转化路径 | — | A→B 有 minimum 30 天承诺 | A→C（短期 hostel 选项）|
| 留存 | — | "all-in 套餐"模式 | 弱 |

**做对了**：
- **Daily events 密度极高** —— 瑜伽、冲浪、烹饪、live music 几乎每天 —— 给 Alex/Mia 不会无聊的承诺
- 4 种房型分级清晰 —— Path B 决策辅助
- "Live · Work · Connect Globally" 三词 tagline —— Path A 第一屏抓得住
- Waitlist 机制 —— scarcity 心理学
- Wellness 课程标准化 —— 可复制的"内容产品"层

**做错了**：
- 嬉皮 luxe 风格审美过度 —— 把不喜欢这种调性的人挡在门外
- "Insta-ready" 设计 → 但成员留存数据据说不强
- 商业上踩雷过（2023 财务问题）—— 重资产模型风险高

**cyc 该偷**：🟢 **每日活动节奏** —— Mia 的"今晚有什么"可以直接答上；Wellness 标准内容产品化；Waitlist 机制
**cyc 该避免**：🔴 重资产 capex 路线 · 单一审美绑定（cyc 三层架构允许多调性共存）

---

### A4. 706 青年空间 🇨🇳

**一句话**：北京海淀起家的青年文化社群，左翼读书会 + 客厅文化 + 全国分舵，最早期的"中国版 Hampshire College"。

| | 对玖玖 | 对 Alex | 对 Mia |
|---|---|---|---|
| 吸引点 | ✅ 高（创意/独立人群对调性认同）| ⚠️ 中（要会左翼语境）| ✅ 中（有故事可发）|
| 转化路径 | 模糊（招募会员制）| 模糊 | 模糊 |
| 留存 | 强（社群文化深）| — | — |

**做对了**：
- **客厅文化** —— "去 706 沙发坐坐"成为一种社交动作 → cyc 的 L3 客厅可以学
- 文化叙事极强 —— 一篇豆瓣长文能讲 10 年故事
- 全国分舵网络 → 异地复用模型（这个对 cyc 的 OS 化战略是直接证据）

**做错了**：
- 网站长期没好好做 —— Path A 入口靠豆瓣 / 微信公众号
- 价格 / 加入路径不透明 —— Path B 完全靠人介绍
- 调性偏单一（左翼青年）—— 排他性

**cyc 该偷**：🟢 **客厅作为社交单位** —— 不是房间不是工位，是"沙发可以坐谁来都行" · 全国分舵 OS 化思路（这就是你 [Selina × Notion] 战略的国内先例）
**cyc 该避免**：🔴 网站长期不做（Path A 完全失守）· 调性单一

### A5. 阿那亚 🇨🇳

**一句话**：秦皇岛海边度假地产改造的"人生可以更美"社区品牌，从地产卖到精神。

| | 对玖玖 | 对 Alex | 对 Mia |
|---|---|---|---|
| 吸引点 | ❌ 不在地理范围 | ❌ 太贵不可持续 | ✅ 强（朝圣式打卡）|
| 转化路径 | — | — | A→C 一次性 |
| 留存 | — | — | 朋友圈传播 |

**做对了**：
- **品牌叙事天花板**："人生可以更美"成为一句通用 meme —— Path A 极致
- 孤独图书馆 + 礼堂 + 戏剧节 → **每个内容都是文化事件**（不只是活动）
- 视觉系统统一性 → Mia 拍照打卡传播力极强

**做错了**：
- 价格门槛极高 —— 普通玖玖 / Alex 不可触
- 重资产，不可在大理短期复用

**cyc 该偷**：🟢 **每个活动是文化事件** —— cyc 的活动通告应该有 Mia 拍照传播的 visual punch · 品牌叙事的"meme 级 tagline"
**cyc 该避免**：🔴 中产消费品牌定位（cyc 的"实验/playful/有点怪"调性正好是反面）

### A6. 安福路 OnTheRoad（上海）🇨🇳

**一句话**：上海安福路上的小型创意 + 社交空间，更像 boutique hub。

**做错了**：单点空间，无 OS 化路径
**对 cyc 启示**：你可以学他们的"街区身份感"，但你已经有"链岛 / 大理"这个更强的地标锚点

### A7-A8. Roam / Sun and Co. 🌍

**一句话**：高端国际 coliving，规模小但精品向。

**做错了**：太精英，不可大众化
**对 cyc 启示**：精品向不冲突，但 cyc 应该用"在地深度"而非"国际精英"作为差异化

## B 组 — 社区软件（留存 / 数字 OS 对标）

### B1. Read.cv 💼

**一句话**：创意人 profile 平台，"个人简历更像作品集"。已被 Perplexity 收购。

| 对 cyc 启示 |
|---|
| ✅ profile 设计的当代典范 —— 真名 + 照片 + 一句话 + 链接 + 时间线 |
| ✅ "无限滚动的极简典型工作经历" 设计语言 |
| ✅ identity 即关系 —— 有 Read.cv profile = 创意圈通行证 |
| ⚠️ 但被收购意味着这个 niche 难以独立持续 → cyc 不应该把"profile 平台"当唯一壁垒 |

**核心 takeaway**：cyc 的成员 profile 应该向 Read.cv 看齐 —— 但定位是"链岛社区身份卡"，不是通用 profile 平台。

---

### B2. Substack 📨

**一句话**：newsletter + 订阅 + 社区 chat 一体化。

**做对了**：
- 单一作者驱动的内容载体 —— 个人故事即产品
- 订阅模式 = 持续付费意愿
- chat 层让付费订阅者有"专属感"

**对 cyc 启示**：
- 🟢 cyc 的活动 / 故事 / 通讯是不是可以变成 substack 化的"链岛 newsletter"，让 Mia 走后还能订阅、Alex 走前先订阅评估？
- 🟢 付费订阅可以是"远程关注但不入住"的中间态变现入口

---

### B3. Discord 🎮

**一句话**：免费 server + 付费 nitro + 重度沉浸社群。

**做对了**：
- 频道分层（话题 / 项目 / 闲聊）—— 信息密度可控
- voice channel 让远程社群有"在场感"
- bot 生态 → community OS 雏形

**对 cyc 启示**：
- 🟢 cyc 的内部社群（L3 客厅）可以借 Discord 的频道分层思路，但**不要变成 Discord** —— 你们的内容更"出版"而非"聊天"
- 🟢 voice channel 思路可以变成"今天谁在 cyc"实时在场指示

---

### B4. Slack 早期 💬

**一句话**：Tiny Speck 内部工具→开放→千亿 SaaS。

**对 cyc 启示**：
- 🟢 **这是你 build sequence 最直接的样板** —— "工具先行验证 → 通用化产品"
- 🟢 Slack 的早期没人想到它会变 SaaS 平台 —— 你也不需要现在就 OS 化，先把工具做到不可替代
- 🟢 留意：Slack 商业化的关键是"团队 = 单位"。cyc 的"单位"是什么？空间？社群？活动？这个值得 Phase 2.5 hypothesis 验证

---

### B5. Notion Community 📝

**一句话**：工具产品 + 自发社区在 Twitter / Discord。

**做对了**：工具本身的强 utility → 社区自发涌现 → 工具公司不需要运营社区
**对 cyc 启示**：
- ⚠️ Notion 模式不适合 cyc —— 因为 cyc **就是社区**，不是工具的副产物
- 🟢 但工具感 / utility-first 这个起点是对的（你的 PLAN.md 也是这个思路）

### 矩阵读出来的三件事

1. **没有任何一个竞品同时打透 A/B/C 三条路径** —— DNA / Outsite 强 B 弱 A，706 / 阿那亚强 A 弱 B，Selina 强 C 但 A 视觉绑死单一审美
2. **profile / 身份层是空地**：实体 coliving 几乎没人做（Read.cv 做了但不是社区）—— **这是 cyc 的明显机会**
3. **OS 化能力是几乎所有人的弱点** —— DNA / 阿那亚都是单点空间，Outsite / Selina 是"重资产复制"。**轻量 OS + 在地深度**是开放空地

## cyc.center 的差异化机会

把 personas + 矩阵叠在一起，差异化清晰：

```
┌────────────────────────────────────────────────────────┐
│ 实体在地（X 轴：地理深度）                              │
│                                                         │
│   阿那亚 ●  706 ●         ← 在地深 / 复用弱             │
│                                                         │
│   DNA ●                                                 │
│              ╲                                          │
│   Selina ●    ╲   ★ cyc.center 目标位                  │
│                ╲   = 在地深 + OS 化轻 + 三 persona 兼容 │
│   Outsite ●     ╲                                       │
│                                                         │
│   ─── 通用工具（Y 轴：OS 化能力） ─────                 │
│                          Read.cv ●  Substack ●          │
│                                  Discord ●              │
│                                            Notion ●     │
└────────────────────────────────────────────────────────┘
```

**cyc 的可防御性来自三件事**：

1. **大理在地真实性**（DNA / 阿那亚 / Selina 都没有的"既有具体地点又能 OS 化的二元性"）
2. **三 persona 同时容纳**（不像 DNA 锁游民、706 锁青年文化、Selina 锁嬉皮 luxe）
3. **profile 横切武器**（实体 coliving 全员空地，Read.cv 已被收购）

## 偷 5 件、避 3 件（行动清单）

### 🟢 立即偷（Phase 3 之前消化）

1. **DNA 的价格梯度透明** → Path B 房型页直接抄它的清晰度
2. **Outsite 的 Membership ≠ 居住** → cyc 的会员可以包括"远程关注的玖玖"和"试住的 Alex"
3. **Selina 的每日活动节奏** → Path C 持续供给给 Mia "今晚有什么"
4. **706 的客厅作为社交单位** → cyc 的 L3 客厅设计的核心隐喻
5. **Read.cv 的 profile 设计语言** → cyc 的成员卡片直接借鉴

### 🔴 必须避免

1. **DNA 的单一身份标签**（"游民"）—— cyc 同时容纳玖玖+Alex+Mia
2. **阿那亚的中产高价定位** —— 反 cyc 的 "实验/playful/有点怪"
3. **Selina 的重资产复制路线** —— OS 化要轻，房子可以租可以合作不要自己建

## 给 Phase 2.5 hypothesis 的种子

矩阵读出的三个最值得验证的假设候选：

**H1**：profile 系统是 3 个 persona 共同的高 ROI 单点（之前已浮现，矩阵进一步确认 —— 因为这是市场空地）

**H2**：cyc.center 的可防御性 = 在地真实性 × OS 化轻量 × 三 persona 容纳 —— 三者必须同时存在，丢一个就掉到现有竞品 lane

**H3**："工具先行 → OS 化"路径，关键里程碑是**第一次给另一个空间授权使用 cyc OS** —— 这个时点 = PMF 信号 = 投资人语言里的"traction"

下一步用 define-hypothesis 把 H1 / H2 / H3 写出 testable 的版本 + 用 measure-experiment-design 设计验证方法。
