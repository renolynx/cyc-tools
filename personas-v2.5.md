## Persona 1 — "玖玖"（在地创意从业者，核心用户）

> *"我从上海跑到大理三年了，不是来躺平的，是想找一群在认真做事的人。"*

### Identity & Context 🟢
- **年龄**：32
- **职业**：独立设计师 + 在大理古城北门开了一间小咖啡馆"角落"
- **来源**：原本上海互联网公司，2023 年搬来大理，现在算半工作半生活
- **居住**：自己在凤仪街租了一间院子，不住 CYC 但路过经常拐进来
- **网络存在**：小红书 1.5k 粉，即刻活跃用户，朋友圈半私密
- **收入**：咖啡店勉强打平，设计单子撑日常，状态："够用、不焦虑"

### 一日切片 🟢
```
07:30  起床，凤仪街溜娃 / 喝茶
09:00  开店，整理一周新到的咖啡豆
11:00  接到客户改稿微信，回到吧台后面打开笔记本
14:00  店里没人，做自己的项目稿
16:30  熟客 A 来聊她想合作市集
19:00  关店，去朋友家吃饭
21:00  刷小红书 / 在某个 30+ 微信群里看到一条 cyc 活动
22:00  报名 → 加日历 → 睡
```

### Motivations（按强度排序）🟢

1. **找到同类** — 其他独立项目主理人，能聊业务也能聊生活
2. **保持创作能量** — 不被本地拖回"佛系慢节奏"刻板印象
3. **延展业务网络** — 找设计师朋友、找合作店铺、找内容渠道
4. **认知刷新** — 接触不同领域的人和想法，避免在地圈层固化

### Jobs to be Done

| 维度 | Job |
|---|---|
| **Functional** | "帮我每周筛掉 80% 不相关活动信息，留下我会真去的那 20%" |
| **Emotional** | "让我感觉我做的小生意也是值得被看见的、不是孤岛" |
| **Social** | "让我在大理本地有一个'看得见的同类圈层'" |

### Pain Points 🟢
- 微信群 30+ 个，活动信息散落各处，**FOMO 又疲惫**
- 在地朋友圈成熟但封闭，新朋友难进
- 想找特定能力的人合作（比如懂内容的）—— 没有结构化途径
- 主理人之间彼此都"听说过"，但没有正式介绍场域

### Tools 用得最多 🟢
微信群 → 小红书 → 即刻 → Notion 个人 → 飞书（自营业务用）→ 偶尔 Substack 订阅

### 与 cyc.center 的关系

| 触点 | 频率 | 主要 Path |
|---|---|---|
| 浏览活动列表 | 🟢 每周 1-2 次 | **Path C** |
| 报名活动 | 🟢 每月 1-2 次 | Path C |
| 看成员 profile | 🟡 想看，但当前没有 | Path C 延伸 |
| 考虑租 cyc 工位 | 🔴 偶尔（自己有店） | Path B 边缘 |
| 自己发起活动用通告生成器 | 🟢 已经在用 | L2 工具间 |

**主路径**：Path C 为主 + L2 工具间高频留存。

### 激活触发（让她从被动到主动）🟡

- **看到具体的人**："今晚有 Sara（她也搞过自营 SKU）来分享供应链坑" → 立刻报名
- **看到自己被记住**：cyc 推送说"上次你来的活动有 3 个新人提到记得你的咖啡店" → 想再参与
- **看到合作机会**：cyc 上线"找搭子"标签 → 玖玖发"找内容合作伙伴" → 真的有人联系

### Anti-fit signals（这不是玖玖时）

- "面向数字游民的 onboarding" → 她会反感（"我不是游民，我在这儿生活"）
- "30 天 community challenge" → 她不会参与（不是来打卡的）
- 收年费会员 → 她可能不付（自己生意没赚多少，时间是稀缺资源不是钱）

### 设计含义

1. **L2 工具间继续是她的留存核心** —— 不要让 L4 公共面的"营销味"污染到她日常用的工具
2. **profile 系统帮她展示"角落咖啡店主理人 + 设计师"双身份**，是她进入合作的钩子
3. **约饭工具 / 找搭子机制**对她价值最大 —— 比"通用社交"更实用
4. **活动通告生成器是她的 day-1 工具**，必须保持不变 —— 这是产品的入口忠诚度

## Persona 2 — "Alex / 王柏"（数字游民 / 远程工作者，融入入口）

> *"我来试三个月，看大理是不是我下一个 base。如果好，我可能不走了。"*

### Identity & Context 🟢
- **年龄**：28
- **职业**：远程产品经理（境外科技公司 / 跨境业务），副业写一个独立工具
- **来源**：从深圳来大理，公司 100% remote，租了 3 个月住处试水
- **居住**：在 Airbnb / 月租公寓，**正在评估要不要长期定居 → 是 cyc 潜在 booking 用户**
- **网络存在**：X / 小红书 / GitHub / 微信都用，跨语境
- **收入**：稳定（远程薪水），生活成本相对宽松

### 一日切片 🟢
```
06:30  起床（北美时差），开 Slack 处理夜里堆的消息
09:00  和北美同事开会到 11 点
12:00  健身 / 出门吃饭
14:00  自己的副业项目，2-3 小时深度工作
17:00  探索大理 / 见朋友 / 找咖啡店
20:00  晚饭 + 社交（找当地人或其他游民）
23:00  小红书刷"在大理生活"内容，看到一篇提到 cyc
23:30  打开 cyc.center —— 第一次访问
```

### Motivations（按强度排序）🟢

1. **找一个生活节奏更对的 base** —— 比深圳/上海更舒服但又能保持工作节奏
2. **建立新的本地社交** —— 不只是其他游民，更想认识本地的人
3. **找副业 / 创业的同类** —— 想从远程打工慢慢过渡到独立
4. **降低决策风险** —— 试住前充分了解社区文化，避免踩坑

### JTBD

| 维度 | Job |
|---|---|
| **Functional** | "帮我在 30 分钟内判断 cyc 是不是我能融入的地方，给我足够细节做长租决定" |
| **Emotional** | "让我感觉我不是来薅本地资源的'游民'，是有可能被欢迎成为成员的" |
| **Social** | "让我能找到第一批可以一起吃饭的本地人 + 同类游民" |

### Pain Points 🟢

- 信息不对称严重 —— 来之前根本不知道大理"哪片好"、"哪个圈层"
- "数字游民"这个标签让本地人有距离感，难破冰
- 大理空间质量参差，价格不透明，**怕踩坑**
- 想长期居住但又怕 commit 太早

### Tools 用得最多 🟢
小红书 → X → 微信 → Discord（远程工作群）→ Notion → Lark / Slack（工作）

### 与 cyc.center 的关系

| 触点 | 频率 | 主要 Path |
|---|---|---|
| 第一次着陆 | 🟢 一次性 | **Path A**（关键转化窗口）|
| 看房型 / 价格 / 试住 | 🟢 至少 2-3 次（决策周期内） | **Path B** |
| 来一场活动试水 | 🟢 1-2 次 | **Path C**（在决策中） |
| 加入决定后 → profile + 留存 | 🟡 取决于前面体验 | L3-L1 |

**主路径**：完整 A → B → C，决策周期 2-4 周。

### 激活触发 🟡

- **Path A**：第一屏看到"做独立项目 / 远程工作的人在大理建的工具站"——他认得出"这是给我做的"
- **Path B**：看到"试住一周 ¥X" 比"加入会员 ¥Y"友好 100 倍
- **Path C**：看到一场"远程产品经理 / 独立开发者聚会"立刻报名
- **决策瞬间**：他已经看了空间相册 + 3 个成员评价 + 试住政策 + 一场活动 → 申请

### Anti-fit signals

- 全是"在地"叙事、没有英文 / 没有跨文化的人 → 觉得是封闭社群
- 写"远离都市内卷"等说教文案 → 他不需要别人定义他为什么来
- 没有透明价格 → 直接关页面（remote 工作者最讨厌"联系我们了解定价"）

### 设计含义

1. **Path A 首屏必须在 10 秒内通过 identity-fit signal** —— 一句话定位 + 第二屏立刻让他认出自己
2. **Path B 必须有透明价格 + 试住承诺**（loss aversion 反向用）
3. **房型 / 评价 / 试住三件套**是他决策路径上的必经环节
4. **profile 系统让他能在还没住进来时就"先认识几个人"**（社交担保 = 决策担保）
5. **支持英文** —— 至少 hero 区一句英文 tagline 体现包容性
6. **介绍朋友机制**：他如果留下来会带朋友 / 同事来 —— referral 设计有 ROI

## Persona 3 — "Mia / 林夏"（短期访客 / 社区脸面）

> *"我朋友圈看到 cyc 一张活动照片，觉得 vibe 对，下个月来玩想找一场参加。"*

### Identity & Context 🟢
- **年龄**：24
- **职业**：北上广深咨询 / 互联网在职年轻人，年休 7 天来大理
- **来源**：朋友圈 / 小红书发现 cyc.center
- **居住**：来大理时住民宿 5-7 天，**完全不会成为长租会员（Path B 不适用）**
- **网络存在**：小红书重度（10+ 篇笔记/年）、朋友圈、抖音少量
- **收入**：固定工资，年假是稀缺资源不是钱

### 一日切片（在大理时）🟢
```
09:00  古城 / 苍山 / 洱海经典路线已 done
12:00  小红书搜"大理 非游客 推荐"
14:00  找到一篇"大理本地人推荐 X 个真实社区"——cyc 在列
14:30  访问 cyc.center → 第一印象决定要不要去活动
16:00  发现今晚有读书会，立刻报名
20:00  到现场，紧张但兴奋
23:00  回民宿，朋友圈发一条"大理找到一个真实社群" + 标 cyc.center
```

### Motivations 🟢

1. **逃离游客身份** —— 不想被认作"打卡的"
2. **真实体验本地** —— 想见到本地人在做什么真事
3. **回去后有内容可发** —— 朋友圈 / 小红书的社交资本
4. **认知拓宽** —— 短暂窥见另一种生活的可能

### JTBD

| 维度 | Job |
|---|---|
| **Functional** | "帮我在 24 小时内找到一场低门槛、能直接走进去的活动" |
| **Emotional** | "让我感觉我不是游客，是被欢迎进去看看的客人" |
| **Social** | "让我有一个值得发朋友圈的故事和照片" |

### Pain Points 🟢

- 旅游攻略上的"必去"她全 done，**找不到非游客活动**
- 怕"打扰本地人"或"显得像猎奇游客"
- 时间窗口窄（5-7 天）—— 必须快速决策
- 怕活动是会员制 / 要预付费 / 不接受外人

### Tools 🟢
小红书（搜索为主）→ 朋友圈 → 大众点评 → 马蜂窝（少）

### 与 cyc.center 的关系

| 触点 | 频率 | 主要 Path |
|---|---|---|
| 搜索 / 朋友推荐发现 | 🟢 一次性 | Path A 入口 |
| 着陆 → 看活动 → 报名 | 🟢 当天内完成全流程 | **Path A → C**（无 B）|
| 来活动 + 拍照发朋友圈 | 🟢 是她的核心动作 | Path C 高潮 |
| 后续关注 / 订阅 | 🟡 走的时候关注一下 | 弱留存 |
| 长租 / 成为会员 | 🔴 几乎不会 | 不走 Path B |

**主路径**：A → C，单次访问，但**口碑传播价值 >> 直接转化价值**。

### 激活触发 🟢

- **Path A**：小红书 / 朋友圈引流 → 落地后第一屏要立刻消解她的顾虑（"这里欢迎来玩的人吗？"）
- **Path C**：活动详情页明确写"对外开放，欢迎大理短期访客"
- **传播触发**：活动后页面有照片墙 + "@cyc.center 标记"提示 → 她拍照发小红书时会标 cyc
- **可能走 Path B 的偶发情况**：来一次后真的爱上 → 半年后辞职来住（少数但存在）

### Anti-fit signals

- 写"会员专属"或"加入社群" → 她秒退
- 要她填表 / 注册 / 验证才能看活动 → 直接走
- 全中文 + 在地话术过重 → 她可能觉得"我是不是听错了"

### 设计含义

1. **Path A 首屏必须友好欢迎短期访客** —— 不只是给会员看的
2. **活动详情页明确"开放欢迎"标签**（"对外开放"/"欢迎来玩"）
3. **活动结束页面是她的传播入口** —— 给她拍照和分享的钩子（"用 #cyc 标记，你的照片可能上社区墙")
4. **不要让她注册才能看任何东西** —— SEO 友好 + 公开可访问
5. **她不付费但她带流量** —— 优化她的体验是 marketing ROI 最高的事之一
6. **接 SEO**：小红书 / Google 搜"大理 真实社区" / "大理 非游客活动"应该能找到 cyc

## 横切总结

### 三个 Persona 的共同点（这是产品 DNA）

| 共同点 | 实现含义 |
|---|---|
| 都讨厌"被运营"的感觉 | 文案不能 SaaS 化，要有人味 |
| 都看重"具体的人 + 真实故事" | profile 系统 + 真名 + 真照片是必需品 |
| 都来自小红书/朋友圈/口碑 | SEO + 社交媒体可分享性是入口 |
| 都对"加入门槛"敏感 | 任何"必须注册才能 X" 都会损失人 |

### 三个 Persona 的关键差异

| 维度 | 玖玖（核心）| Alex（融入）| Mia（脸面）|
|---|---|---|---|
| 关心的最终问题 | "我能找到合作的人吗" | "这是不是我下一个 base" | "今晚有什么可以去的" |
| 决策时间窗 | 持续（已在地） | 2-4 周 | 24 小时 |
| 关键 Path | C + L2 | A→B→C 全程 | A→C |
| 主要价值 | 留存 + 主理人网络 | booking 转化 | 口碑 + SEO |
| 不可替代的设计 | 找搭子 / profile / 通告生成器 | 透明价格 / 试住 / 评价 | 开放活动 / 拍照入口 / 不强制注册 |

### 我的最终判断 → 产品决策

**做对所有 3 个 persona 的最高 ROI 单点改造**：

1. **profile 系统**（横切）—— 玖玖用来展示自己 / Alex 用来评估社区 / Mia 用来看发起人是谁
2. **活动详情页改造**（横切）—— 加"对外开放"标签（Mia）+ 已报名头像（Alex）+ 发起人 profile 链接（玖玖）
3. **Path A 首屏 v1**（Alex + Mia 必需）—— 玖玖虽然不需要但她路过看到也不会被劝退

→ **这三件事 = Phase 3 第一周 unlock 的全部**。
