00 · Intro

写在前面

我们是什么

CYC 链岛（主页：cyc.center）不是常规酒店，也不是普通民宿。

  • 空间：大理在地 17 间房 + 公共客厅 / 厨房 / 学习室 / 活动场所
  • 业主：社区。这家酒店通过社区众筹共建产生，所有权和重大决策权属于 CYC 社区方
  • 客群定位：从"随机游客"过渡到"长期价值住客"——能参与社区共建、文化共创、资源置换的年轻人
  • 商业逻辑：住房保本，二销赚——真正的利润来自活动、文化产品、资源置换、社区共创
  • 现有团队：在地长期主理人 + 咖啡店老谭 + 社区代表 + 自媒体 + 财务

简单说：我们既要让酒店稳定赚钱（基础），也要让它成为社区凝聚和文化共创的载体（差异化）。

为什么这一轮招人

7-8 月是大理旅游旺季，也是我们必须把"基础正常态"跑通的窗口。6 月底之前必须组建好团队，否则旺季流量接不住。

这一轮招的 4 个岗位（店长 / 副店长 / 客房保洁 / 住宿义工 ×2）覆盖酒店的"基础正常态"——把房卖好、把客接好、把房间打扫干净、把成本控住、把住客体验做稳。

社区化改造（活动策划、客户筛选、文化营造）由 CYC 社区团队 + 酒店团队联合推动——所以即使你应聘的是酒店岗位，也会接触到社区营造这一面。这是 CYC 与普通民宿酒店最大的差别。

我们的设计原则

这份招募方案不是凭空想出来的，是基于 CYC 过去 4 个月运营踩过的坑总结出来的：

  1. 激励机制要引导正确行为——纯靠 OTA 降价冲入住率是死路，所以薪酬结构按客源渠道差异化设计
  2. 必须有持续抽查 + 落地奖惩——没有抽查标准必然漂移，没有奖惩主动性必然下降
  3. 社区必须在汇报链里——这家酒店是社区的，不是物业团队自己的小王国
  4. 职业化的事交给职业团队，但他们必须服务社区——这是合同条款，不是文化期待
  5. 创造性工作要有出口和度量——社区自媒体是每个岗位的必选项，不是"宣传部门的事"

对外的宣发文案要回答清楚 3 个问题，让对的人自己来：

  1. 这是什么样的地方？
  2. 这里都是什么样的人？
  3. 对这里可以有什么期待？

这两条战略和下面的团队招募方案是同一件事的两面。

战略一（商业基础）需要稳定的酒店团队来执行——这是接下来这份招募文档的目的；
战略二（社区定位）需要每个团队成员理解并参与——这是后续岗位 JD 中"服务社区条款"和"社区自媒体"的来源。

03 · Position

岗位 JD · 店长
Role 01 · 经营负责人

店长 Hotel Manager

也可写作：运营负责人 / 民宿主理人
全年平均≈ ¥7,000/月
岗位定位：酒店的经营负责人。对收入、成本、利润、服务体验、团队稳定、空间秩序整体负责。
一句话区分：副店长负责每天把事情做好，店长负责让整个系统持续赚钱、稳定、可复制——并且持续抽查执行层不漂移。

核心职责

1.经营与收入
  • 制定月度入住率 / 营收 / 利润目标
  • OTA 平台（携程、美团、飞猪、Booking、Airbnb 等）的上架、定价、优化
  • 重点：主动减少对 OTA 的依赖，做社区导流 / 直订 / 老客回头
  • 淡旺季 / 房型动态调价；每周复盘渠道表现
2.房态与体验
  • 房态、入住、退房系统的建立和维护
  • 入住体验设计（欢迎信息、社区公约、公共区域说明）
  • 客诉处理与突发事件应对
3.团队管理与抽查
  • 直接管理副店长，分配每日 / 每周任务
  • 每周一次抽查副店长的房态记录、库存盘点、资产登记完整性
  • 每月一次抽查保洁的清洁标准（随机 4 间房 + 1 次公区）
  • 设计义工的职责边界、考核机制
4.标准与 SOP
  • 入住 / 退房 / 客诉 / 房检 / 清洁验收 / 义工培训 / 突发事件等流程的书面化
  • 让酒店运转不依赖个人临场发挥，而是靠系统稳定
5.成本与物资
  • 固定成本（房租 / 物业 / 折旧）和日常消耗（清洁 / 床品 / 耗材）的管控
  • 大额采购决策、供应商对接
6.与社区团队协作（CYC 特色，写进合同）
  • 接收社区代表的方向反馈，把"客户筛选 / 社区营造"翻译成具体运营动作
  • 月度联席会议（与社区主理人 + 社区运营）
  • 联合策划二销项目（活动、共创、资源置换、长住会员制）
  • 不得阻断社区团队与义工 / 住客的直接接触
7.社区自媒体（个人产出 + 团队 KPI）
  • 个人产出：朋友圈 / 视频号片段周 3 条+，视频号成片月 1-2 条，公众号月 1 篇，小红书周 1 篇
  • 从经营 / 行业 / 系统视角输出（行业观察、运营复盘、社区经济思考）
  • 团队 KPI：统筹副店长 + 义工的内容产出，保证团队月度发布基线达标
  • 把"自媒体来源转化"作为渠道结构的一部分纳入月度复盘

必须
  • 有酒店 / 民宿 / 青旅 / 空间运营 / 门店管理经验（至少 1 年管理岗）
  • 能管理人，也能处理具体琐事

  • 有责任心，能兜底，不甩锅

  • 有培训兼职 / 义工 / 短期工的经验
  • 会用飞书、Notion、Excel 等基础数字工具
  • 会做内容（小红书、朋友圈、抖音）

薪酬结构

项目金额 / 比例说明
底薪¥3,500 / 月保底生活线
社区直订 / 老客抽成25%主激励
OTA 渠道抽成10%引导减少 OTA 依赖
OTA 月度占比超 40% 部分5%不是 0，是"不划算"
长期社区会员 / 包月30%最高激励
年终红包视全年经营非合同义务

预期到手：淡季 ¥5,000–7,000 · 旺季 ¥9,000–13,000 · 全年平均 ≈ ¥7,000

为什么这么设计
我们过去试过"纯抽成 30%"模式，结果是管理者依靠 OTA 降价冲入住率，而 OTA 本身要抽 15%，最后被两层抽完，社区拥有酒店但赚不到钱。

这次的设计：同样一单利润 ¥1,000，从社区直订来你拿 ¥250，从 OTA 来只拿 ¥100；OTA 月度占比超过 40%，超出部分只拿 ¥50。我们用激励差直接告诉你：做社区导流远比做 OTA 划算。

考核指标

月入住率 月利润 OTA 评分 月度客源结构 客诉响应 团队稳定度 SOP 执行度 社区协同 自媒体团队 KPI

岗位 JD · 副店长
Role 02 · 现场执行

副店长 Assistant Manager

也可写作：前厅主管 / 运营主管 / 民宿管家
全年平均≈ ¥5,200/月
岗位定位：酒店日常运营的现场执行负责人。统筹待人接物、库存管理、资产管理、保洁协调、义工协调——是酒店每一天能不能正常运转的关键人物。
一句话区分：店长负责经营系统，副店长负责每天让这个系统正常运转。

核心职责（10 类）

1.前台接待（待人接物）
  • 到店接待、入住登记、房间介绍
  • 介绍公共区域（Wi-Fi、热水、厨房、洗衣、学习室）的使用
  • 处理电话、微信、OTA 咨询；说明社区公约
  • 客户筛选的一线判断（按店长 + 社区代表共同制定的标准）
2.订单与房态
  • 每日房态表更新、入住 / 退房 / 续住的确认
  • 避免漏单 / 重复 / 房型错误
  • 每天与保洁对接可入住房间
3.退房与查房
  • 退房提醒、查房、床品更换
  • 房间损坏 / 缺失物品 / 异常的登记和上报
4.清洁协调
  • 每天与保洁对接退房清洁任务
  • 验收清洁质量（基础卫生、床品、异味、死角）
  • 公共区域基础卫生检查
5.库管（重点）
  • 高频耗材的入库 / 出库 / 盘点 / 短缺预警（卫生纸、洗护用品、床品耗材、清洁剂等）
  • 每周一次基础盘点，向店长提采购申请
  • 建立最低库存提醒；避免重复采购 / 闲置 / 丢失
6.资产管理（重点）
  • 房间及公共区域固定资产清单的维护
  • 定期检查资产状态，记录损坏 / 老化 / 待维修
  • 新购资产登记 + 配合店长月度盘点
7.义工管理（运营任务侧）
  • 协调 2 名免费住宿义工的运营任务排班
  • 检查义工运营任务的完成度
  • 创造性贡献的部分由社区团队直接 coach，不由副店长管
8.公共空间维护
  • 维护前台、客厅、学习室、厨房等区域基本秩序
  • 噪音、卫生、公共空间占用的初步沟通
  • 晚间公共区域音量管控
9.日常记录与汇报
  • 每日入住 / 退房 / 客诉 / 维修 / 库存 / 资产变动的记录
  • 每天向店长做简单运营汇报
  • 配合店长每周的抽查
10.社区自媒体（日常出口）
  • 个人产出：朋友圈日更（前台接待手机 9k 好友是主战场），视频号成片月 2 条，公众号月 1 篇，小红书周 2 篇
  • 从待人接物 / 日常服务 / 客人故事视角输出（今日入住、空间状态、暖心瞬间、活动幕后）
  • 协调义工内容产出节奏，避免重复 / 缺位

07 · Community

与 CYC 社区的工作配合

这是 CYC 与普通酒店 / 民宿最大的不同，每个岗位都需要理解。

业主结构

这家酒店是 CYC 社区通过众筹共建的资产。所以：

  • 所有权人：社区众筹方
  • "老板代表"：社区代表（一位 CYC 社区团队的指定人员）
  • 业务方：酒店团队（你应聘的这 4 个岗位）

社区代表不参与日常运营，但在以下事项上是决策方或共同决策方：

  • 客户筛选标准
  • 价格策略大方向
  • 公共空间用途
  • 二销策略（活动收入、内容产品、资源置换的分账机制）
  • 重大投资决策
  • 客源渠道结构（OTA / 直订 / 社区 / 老客的比例红线）

月度联席会议（强制机制）

店长 + 副店长 + 社区主理人 + 社区运营 每月开一次。

议程模板：

  1. 上月经营数据复盘（入住率 / 利润 / 客源结构 / 客诉）
  2. 上月抽查结果（副店长、保洁、义工）
  3. 社区方反馈（客户筛选、活动配合、二销机会）
  4. 下月联合策划（活动、内容、长住客户开发）
  5. 团队问题与调整

二销联动

CYC 的核心商业逻辑是"住房不赚钱，二销赚"。也就是说：

  • 房费保本（旺季有利润）
  • 真正的利润来自：活动门票、文化产品、资源置换、社区会员、共创项目分润等

店长需要和社区团队联合策划二销项目（在地艺术家驻留 + 展览门票 / 文化活动 / 工作坊 / 联合咖啡店餐饮户外的资源置换 / 长住客户的社区会员制）。副店长在这一块的角色是支持执行，不需要独立策划。

客户筛选

CYC 不接所有客人。筛选标准由社区团队和店长共同制定。基本方向：

拒绝
  • 放纵 / 发泄 / 噪音 / 邻里风险型客群
欢迎
  • 创造力 / 文化共创 / 长住 / 学习 / 远程办公 / 艺术家 / 工作坊参与者

08 · Media

社区自媒体 · 创造性工作的主要出口

为什么这是核心

CYC 的核心商业逻辑是"社区导流替代 OTA"。社区自媒体不是"宣传部门的事"——它是每一个酒店团队成员的工作。

  • 对店长：自媒体是"社区导流"这条线的核心 KPI，直接关系到客源结构和你的抽成（OTA 客源抽成 10%，社区直订 25%——自媒体就是把客人从 OTA 拉到直订的桥）
  • 对副店长 / 义工：自媒体是创造性工作的主要表达出口和结果指标。创造性贡献如果没有出口、没有度量，就只是"心意"，不是结果
核心要求：每个值班者都发，发不一样视角的内容。
