## ① Problem（top 3 痛点 + 现有替代品）

### 玖玖型痛点（🟢 高置信度，访谈 + 自身）
1. **活动信息散落在 30+ 微信群**，FOMO 又疲惫
2. **找合作伙伴没有结构化途径** —— 听说过但没正式介绍
3. **小生意/项目缺被看见的渠道** —— 朋友圈 reach 衰减

### Alex 型痛点（🟡 推理 + 部分访谈）
1. **来大理前严重信息不对称** —— 不知道哪片好、哪个圈层适合
2. **怕踩坑高 commitment** —— 长租前缺试用机制
3. **本地圈层难破冰** —— 游民标签反而是负资产

### Mia 型痛点（🔴 主要推理，需访谈）
1. **找不到非游客活动** —— 攻略上的"必去"已 done
2. **怕被看作"猎奇"打扰本地** —— 期望明确欢迎信号
3. **体验缺乏 instagram-able 钩子** —— 想发但没场景

### 现有替代品（用户当前怎么解决这些痛点）
- 微信群（信息泛滥）· 小红书（搜索 + 笔记）· 朋友介绍（覆盖窄）
- 实体替代品：DNA 数字游民公社 · 706 青年空间 · Selina · Outsite
- 工具替代品：Notion 个人 · Discord · Read.cv（个人 profile）

→ **没有任何替代品同时解决 3 个 persona 的痛点**。这是 cyc.center 的存在理由。

## ② Customer Segments

### 主要（核心服务对象）🟢
**大理本地创意/独立从业者**（玖玖型）
- 数量级估计：大理常住的此类人 1000-3000（待验证）
- 已活跃接触者：估计 50-200（飞书群、活动到访过）
- 关键特征：已厌倦"被运营"、看重真实人事、对"游民标签"反感

### 次要（拉新增长）🟡
**远程工作者 / 数字游民**（Alex 型，评估期）
- 数量级：每年来大理短期居住的此类人 5000-10000
- 转化窗口：2-4 周决策周期
- 关键特征：对透明度 + 试用机制敏感

### 长尾（流量入口 / 口碑）🟡
**短期访客**（Mia 型）
- 数量级：每年来大理度假的中产 5-7 天访客 100k+
- 单次价值低但传播价值高（小红书一篇笔记可能带 100 个 Alex）
- 关键特征：低门槛 + 拍照传播友好

### 终极（OS 化阶段，6-12 月）🔴
**其他社区空间运营者**
- 数量级：国内类 cyc 空间 30-50 个（706 全国分舵 + DNA + 安福路 + 各地小型）
- 关键特征：想复用模式但缺工具/方法论

### 早期采用者画像（最先付费/最先深度使用）
**"角落咖啡店主理人 Sara"** —— 32 岁、独立项目、已在大理 2-3 年、已活跃在多个微信群、做过 1-2 次自营市集、想找设计/内容合作伙伴 —— **这就是玖玖**。她的画像清晰到能在脑里指认出 5 个具体名字。

## ⑦ Cost Structure

### 固定 / 极低 🟢
- Vercel（hobby tier）+ Upstash KV + 飞书 API：≈ 0-200/月
- 域名：≈ 100/年
- daybreak-os / 设计资产：边际成本 0（已有框架）

### 主要成本：你的时间 🟢
- 产品开发：每周 20-40 小时（vibe coding）
- 社区运营 / 活动主持：每周 10-20 小时
- 内容创作（公众号 / 小红书）：每周 5-10 小时

### 变动 / 高风险 🟡
- **实体空间介入**（如果你做 R1 居住付费）—— capex 高、风险高
- 法务 / 财务 / 注册：融资期一次性 5-20k
- 设计 / 摄影外包：可选，建议先用 AI + DIY

### 烧钱 vs 自给平衡点 🔴
- 假设当前每月烧（你的 opportunity cost）≈ 等同上海 PM 月薪 30-50k
- 收入需要打平：每月活动 + 合作 ≥ 30k（待验证）
- 真正的钱投入：靠融资 / 战略合作 / 你自己积蓄

## ⑧ Key Metrics

### 北极星
**月活跃成员数 + 通过 cyc 入口新成员比例 + 第三方空间 OS 询问次数**

### Activation（关注 6 个月）
- 玖玖型：月度活动出席率（≥ 1 场/月 = activated）
- Alex 型：从首次访问到 RSVP 第一场活动的时间（≤ 14 天 = activated）
- Mia 型：单次访问内 RSVP 一场活动率（首访转化）

### Retention
- 30 天回访率 / 90 天回访率
- 月度活跃成员留存（cohort 分析）
- "标 #cyc 朋友圈/小红书"次数（替代社交分享指标）

### Referral
- 新成员来源占比：cyc 主动 vs 朋友介绍 vs 微信群
- 单个老成员邀请新成员数

### Revenue（按 stream 分）
- 活动付费总额 + ARPU
- 居住占用率（如果做实体）
- Newsletter 订阅数 + 付费订阅率（如果做）
- OS 询问 → 转化授权数（leading PMF 信号）

### 反 KPI（反指标）
- "为了 KPI 而办的活动数"——必须为 0
- 成员之间互动是否变成"任务"——感受性观察

## ⑨ Unfair Advantage（真正不可复制的）

### 已有 🟢
1. **创始人本身是目标用户**（玖玖型）+ 在地（大理 2+ 年）
   - 你的痛点 = 用户痛点 → 不需要 user research 也能基本对路
   - 在地真实关系网络 → 其他玩家进不来
2. **设计 + 工程一体**（RCA + Vibe coding）
   - 极少创始人能同时把视觉做到 daybreak-os 这种水准 + 自己写代码
   - 节省 2-3 个全职岗位的成本
3. **完整 AI toolkit**（85 skills + 3 MCP + 自定义 design system）
   - 单人开发速度 ≈ 5-8 人小团队
   - 这点不对外讲，但执行层面碾压

### 累积中 🟡
4. **链岛社区已有的真实网络**（玖玖型现有圈层）
5. **大理在地的"被信任度"**（连续做事 = 累积信誉）

### 未实现但是潜在 🔴
6. **OS 化方法论文档**（如果做出来 = 招募/复用工具）

## 关键洞察 + 战略选择题

### 洞察 1：Lean Canvas 浮现的"真正杠杆"

按重要度从高到低：
1. **创始人 / 用户对位**（你 = 玖玖）—— 最稀缺、最不可复制
2. **在地真实关系**（大理 2+ 年沉淀）—— 复制需要时间，无法跳过
3. **设计 + 工程一体**（Vibe coding）—— 单人节奏比团队快
4. **多 persona 同时容纳能力** —— 产品架构带来的差异化

### 洞察 2：必须做的战略选择题

#### 选择 A：你要不要亲自介入实体空间运营？
- **如果 Yes**：R1 居住付费成为最大收入流，cost structure 重，但 Selina-style 模式成立
- **如果 No**：你做"工具 + 内容 + OS"，把空间外包给合作伙伴 —— Notion 模式成立

→ 这是**最关键的战略选择**。Phase 4 投资人 demo 之前必须明确。我赌你倾向 **No** —— 你的优势是设计 + 工程 + 内容，不是空间运维。

#### 选择 B：to-C 还是 to-B 优先？
- **to-C 优先**（订阅 / 活动 / 居住）：先把单个空间做透，证明 PMF
- **to-B 优先**（OS 授权）：直接做"给其他空间的产品"，规避空间运维

→ 推荐 **to-C 优先 → 验证 PMF → 再 to-B**。这是 Slack / Notion 路径。

### 洞察 3：3 个最该测的假设

把 lean canvas 转成 hypothesis（Phase 2.5 直接用）：

**H-Canvas-1**：profile 系统上线后，月活跃成员留存率从 X% 提升到 Y%（验证 ④ 解决方案对 ① 痛点的有效性）

**H-Canvas-2**：Path A 首屏改造后，从首次访问到 RSVP 的转化率从 X% 提升到 Y%（验证 ⑤ Channels 漏斗的真实效果）

**H-Canvas-3**：60 天内有 1 个其他空间询问"用 cyc 模式" → OS 化路径成立（验证 ② Customer 终极段 + ⑥ R4 收入流）
